Manyverse (Scuttlebutt)

This is the least straightforward to get started with but is easy enough to use once it's up and running and is currently my main form of social media. It provides similar functions to Facebook but without video.

Scuttlebutt is decentralised to a greater degree than the other entries on this page: there are types of server that exist but users can also exchange posts and messages directly by being on the same WiFi network. The instructions below will connect you to a room server, which we will be using as a simple relay.

This procedure assumes we already know each other.

  • Step 1: Install Manyverse on your phone or computer.
  • Step 2: Open your own profile, click the grey code, select either "Copy SSB URI" or "Copy cypherlink", and send the SSB URI or cypherlink to me via whatever channel we already use to communicate.
  • Step 3: Arrange a time with me for the initial syncrhonization
  • Step 4: At the agreed upon time use the web browser on the same device as Manyverse to open jeremylist.scutttle.space, let the page redirect to Manyverse, and follow the profile that opens. If it says I'm offline then go back to the web browser and click the "Connect with jeremylist" button again. Once the connection has succeeded once it will happen again automatically.

Element (Matrix)

Matrix and the remaining entries on this page are examples of federated systems. This means users connect to servers and user accounts are tied to the servers on which they were created, but users who have chosen different servers are still able to communicate.

Matrix is a federated instant messaging system, and Element is an application that can connect to Matrix.

  • Step 1: Install Element
  • Step 2: Go through the account setup procedure
  • Step 3: Type @j_p_list:matrix.org into the search bar

Mastodon

Mastodon is a federated microblogging network which can be considered a replacement for Twitter.

  • Step 1: Pick a server and create an account. instances.social can help you choose one.
